Regles de contribution du projet.

Ce document etablis par Flora et Lola definis les regles de contribution au projet Clinique Plus

Convention de nommage: Nom de branch
    dev
        prenom

Convention de nommage : Noms des commits :
    <Feat> Ajout d'une nouvelle fonctionnalite
    <Fix> Correction des bugs
    <Docs> Mise à jour ou creation de documents

Regles obligatoires pour les pull requests:
• une branche par developpeur ;
• une Pull Request par ticket ;
• pas de push direct sur main ;
• pas de push direct sur develop, sauf consigne explicite ;
• besoin de validation avant un merge sur dev par flora
